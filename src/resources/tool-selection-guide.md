# Tool Selection Guide for Orchestration Execution

This guide helps AI agents select the appropriate Exa tools when executing orchestration plans generated by the orchestration reasoning system.

## Available Tools by Information Type

### 📰 Current Events & News
**Tool**: `web_search_exa`
- **Use for**: Breaking news, current events, general information, recent developments
- **Example queries**: "latest developments in AI", "recent Tesla news", "what happened today in tech"
- **Strengths**: Broad coverage, real-time information, general purpose

### 📚 Academic & Research
**Tool**: `research_paper_search_exa`
- **Use for**: Scientific studies, peer-reviewed papers, academic evidence, research findings
- **Example queries**: "studies on climate change", "research on neural networks", "academic papers about quantum computing"
- **Strengths**: Authoritative sources, scientific rigor, citation-worthy content

### 🏢 Business & Corporate
**Tool**: `company_research_exa`
- **Use for**: Company profiles, business information, corporate data, organizational details
- **Example queries**: "OpenAI company information", "Tesla financial data", "Microsoft leadership team"
- **Strengths**: Comprehensive business data, financial information, company insights

### 🎯 Competitive Analysis
**Tool**: `competitor_finder_exa`
- **Use for**: Finding competitors, market positioning, industry comparisons, competitive landscape
- **Example queries**: "competitors of Stripe", "companies similar to Airbnb", "Tesla market position"
- **Strengths**: Market analysis, competitive intelligence, industry insights

### 💼 Professional Networks
**Tool**: `linkedin_search_exa`
- **Use for**: Professional profiles, employee information, business connections, career data
- **Example queries**: "CEO of OpenAI", "data scientists at Google", "marketing professionals in NYC"
- **Strengths**: Professional backgrounds, company employees, networking data

### 📖 Factual Information
**Tool**: `wikipedia_search_exa`
- **Use for**: Established facts, historical context, encyclopedic knowledge, definitions
- **Example queries**: "history of artificial intelligence", "definition of machine learning", "overview of quantum physics"
- **Strengths**: Reliable baseline information, historical context, well-structured content

### 💻 Technical Resources
**Tool**: `github_search_exa`
- **Use for**: Code examples, open source projects, technical implementations, developer resources
- **Example queries**: "React authentication examples", "Python machine learning libraries", "Rust web frameworks"
- **Strengths**: Actual code, implementation details, developer documentation

### 💬 Community Sentiment
**Tool**: `scrape_reddit_exa`
- **Use for**: User opinions, community discussions, grassroots sentiment, public perception
- **Example queries**: "what people think about GPT-5", "Reddit discussion on Tesla Cybertruck", "community feedback on new iPhone"
- **Strengths**: Unfiltered opinions, real user experiences, trending discussions

### 📹 Video Content
**Tool**: `youtube_search_exa`
- **Use for**: Video tutorials, visual explanations, demonstrations, educational content
- **Example queries**: "how to use Docker tutorial", "machine learning explained", "React hooks demonstration"
- **Strengths**: Visual learning, step-by-step guides, practical demonstrations

### 🎵 Trending & Viral
**Tool**: `tiktok_search_exa`
- **Use for**: Viral trends, youth perspectives, emerging cultural phenomena, short-form content
- **Example queries**: "trending AI discussions", "viral tech trends", "Gen Z views on climate change"
- **Strengths**: Cutting-edge trends, youth sentiment, viral content

### 🔗 Direct Content Extraction
**Tool**: `crawling_exa`
- **Use for**: Extracting full content from specific URLs when you already know the source
- **Example usage**: When you have a specific article URL and need its complete content
- **Strengths**: Complete content extraction, detailed analysis of known sources

## Execution Best Practices

### 🚀 Parallel Execution
When the orchestration plan allows, execute multiple searches simultaneously:
- Group independent searches together
- Run different tool types in parallel
- Maximize throughput for time-sensitive tasks

### 💾 Cache Considerations
- Similar queries may return cached results
- Vary search terms slightly for fresh perspectives
- Add temporal qualifiers ("latest", "2025", "recent") for current information

### 🔄 Fallback Strategies
If a tool fails or returns insufficient results:
1. Try a more general tool (e.g., `web_search_exa` as fallback)
2. Rephrase the query with different terms
3. Break complex queries into simpler parts

### ✅ Quality Verification
- Check result relevance against the orchestration's quality requirements
- Verify source credibility for high-stakes information
- Cross-reference findings across multiple tools when accuracy is critical

### ⏱️ Timing Optimization
For time-sensitive orchestrations:
- Limit result counts (use smaller `numResults`)
- Prioritize faster tools for initial exploration
- Use targeted queries rather than broad searches

## Tool Selection Decision Tree

```
Is the information about...
├── Current/Recent Events? → web_search_exa
├── Academic/Scientific? → research_paper_search_exa
├── A Specific Company? → company_research_exa
├── Market Competition? → competitor_finder_exa
├── Professional People? → linkedin_search_exa
├── Established Facts? → wikipedia_search_exa
├── Code/Technical? → github_search_exa
├── Public Opinion? → scrape_reddit_exa
├── Video Explanation? → youtube_search_exa
├── Trending Topics? → tiktok_search_exa
└── Known URL? → crawling_exa
```

## Integration with Orchestration Primitives

### Querying Primitive
- Select tools based on information type needed
- Consider using multiple tools for comprehensive coverage
- Balance breadth vs. depth based on orchestration goals

### Filtering Primitive
- Apply relevance thresholds (e.g., confidence > 0.7)
- Filter by date for time-sensitive information
- Remove duplicates across multiple tool results

### Aggregation Primitive
- Group results by source type
- Identify common themes across tools
- Synthesize contradictory information

### Reasoning Primitive
- Use filtered and aggregated data as evidence
- Apply domain-specific reasoning patterns
- Generate insights that go beyond raw search results
